## Notes for downstream consumption

The output's structure is deliberate:

- **`### Step N` headings** are scannable by downstream skills as epic boundaries — a regex over the headings yields the step list without parsing prose.
- **The frontmatter** carries the metadata (date, input PRD, step count) downstream skills may want.
- **The "Negotiation history" table** preserves user intent — if sprint planning later asks "why is X before Y?", the answer is in this table.
- **The "Out of scope" table** prevents downstream skills from accidentally pulling in capabilities the user has chosen to defer.

This artifact is the single source of truth for execution sequencing once written. Re-running this skill regenerates it; manual edits are fine but should be re-run through the skill if the changes are substantive enough that the dependency graph shifts.
